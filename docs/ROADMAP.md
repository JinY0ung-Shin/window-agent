# Window Agent - AI 회사 운영 시스템

## 프로젝트 개요

**Window Agent**는 사용자가 가상의 회사를 운영하듯 여러 AI 에이전트(직원)에게 업무를 지시하고,
에이전트들이 실제 Windows PC에서 작업을 수행하는 데스크톱 애플리케이션이다.

사용자 = **CEO**, AI 에이전트 = **직원**

---

## 핵심 컨셉

### "나만의 AI 회사"

```
┌─────────────────────────────────────────────────┐
│                  Window Agent                    │
│                                                  │
│   👤 CEO (사용자)                                │
│    ├── 📋 비서실 ── 비서 에이전트               │
│    ├── 💻 개발팀 ── 개발자 에이전트              │
│    ├── 📊 분석팀 ── 분석가 에이전트              │
│    ├── 📝 기획팀 ── 기획자 에이전트              │
│    ├── 🔍 조사팀 ── 리서처 에이전트              │
│    ├── 🎨 디자인팀 ── 디자이너 에이전트          │
│    ├── 📁 관리팀 ── 파일/시스템 관리 에이전트     │
│    └── 🔧 IT팀 ── 자동화/스크립트 에이전트       │
│                                                  │
│   [회사 대시보드] [채팅] [업무 보드] [보고서]     │
└─────────────────────────────────────────────────┘
```

### 운영 흐름

1. **CEO가 지시** → "이번 달 매출 보고서 만들어줘"
2. **비서가 분배** → 분석팀에 데이터 분석 요청, 기획팀에 보고서 포맷 요청
3. **직원들이 협업** → 각자 역할에 맞는 작업 수행 (파일 읽기, 프로그램 실행, 웹 검색 등)
4. **결과 보고** → CEO에게 완성된 보고서 제출

## 에이전트(직원) 정의

### 기본 에이전트 구성 (8명)

#### 1. 비서 (Secretary) - "김비서"
- **역할**: CEO 지시 해석, 업무 분배, 일정 관리
- **능력**: 자연어 이해, 태스크 분해, 에이전트 간 조율
- **특징**: 모든 지시의 첫 번째 수신자, 다른 에이전트에게 업무 위임

#### 2. 개발자 (Developer) - "박개발"
- **역할**: 코드 작성, 스크립트 실행, 기술 문제 해결
- **능력**: 코드 생성, 파일 편집, 터미널 명령 실행, exe 프로그램 실행
- **도구**: 파일 읽기/쓰기, Shell 실행, 프로세스 관리

#### 3. 분석가 (Analyst) - "이분석"
- **역할**: 데이터 분석, 통계, 차트 생성
- **능력**: CSV/Excel 파일 분석, 데이터 시각화, 통계 요약
- **도구**: 파일 읽기, Python 스크립트 실행, 차트 생성

#### 4. 기획자 (Planner) - "최기획"
- **역할**: 문서 작성, 기획서 작성, 보고서 포맷팅
- **능력**: 문서 생성, 마크다운/워드 변환, 프레젠테이션 생성
- **도구**: 파일 쓰기, 문서 변환 도구

#### 5. 리서처 (Researcher) - "정조사"
- **역할**: 웹 검색, 정보 수집, 요약, 사내 시스템 데이터 수집
- **능력**: 웹 크롤링, 검색, 뉴스 모니터링, 정보 요약, 로그인 필요한 SaaS 접근
- **도구**: 웹 브라우저 자동화, 검색 API, 스크래핑
- **접근 가능**: Confluence, Notion, Jira, Google Docs 등 (브라우저 자동화)

#### 6. 디자이너 (Designer) - "한디자"
- **역할**: 이미지 생성, UI 목업, 시각 자료 제작
- **능력**: 이미지 생성 AI 활용, SVG 생성, 간단한 편집
- **도구**: 이미지 생성 API, 파일 쓰기

#### 7. 시스템 관리자 (SysAdmin) - "강관리"
- **역할**: 파일 정리, 시스템 모니터링, 백업 관리
- **능력**: 파일 시스템 조작, 프로세스 관리, 디스크 관리
- **도구**: 파일 시스템 API, Shell 명령, 시스템 정보 조회

#### 8. 자동화 전문가 (Automator) - "윤자동"
- **역할**: 반복 작업 자동화, 매크로 생성, 스케줄링
- **능력**: 스크립트 작성, 크론 작업, 프로그램 자동 실행
- **도구**: 스케줄러, Shell, 프로세스 실행

### F4. 도구 시스템 (Tool System)

에이전트가 실제 Windows PC에서 작업을 수행하기 위한 도구들.

#### 4-1. 파일 시스템 도구
| 도구 | 설명 |
|------|------|
| `file.read` | 파일 읽기 (텍스트, CSV, JSON 등) |
| `file.write` | 파일 쓰기/생성 |
| `file.list` | 디렉토리 내용 조회 |
| `file.search` | 파일 검색 (이름, 내용) |
| `file.move` | 파일 이동/이름 변경 |
| `file.delete` | 파일 삭제 (확인 필요) |
| `file.zip` | 압축/해제 |

#### 4-2. 프로세스 실행 도구
| 도구 | 설명 |
|------|------|
| `process.run` | exe/프로그램 실행 |
| `process.shell` | Shell 명령 실행 (PowerShell, CMD) |
| `process.list` | 실행 중인 프로세스 목록 |
| `process.kill` | 프로세스 종료 (확인 필요) |
| `process.schedule` | 예약 실행 |

#### 4-3. 웹 도구 (단순 HTTP)
| 도구 | 설명 |
|------|------|
| `web.search` | 웹 검색 |
| `web.fetch` | 웹 페이지 내용 가져오기 (REST API, 정적 페이지) |
| `web.download` | 파일 다운로드 |

#### 4-3b. 브라우저 자동화 도구 (Chrome 제어)
| 도구 | 설명 |
|------|------|
| `browser.open` | 브라우저에서 URL 열기 |
| `browser.login` | 로그인이 필요한 사이트 인증 처리 |
| `browser.screenshot` | 페이지 스크린샷 캡처 |
| `browser.extract` | JS 렌더링 페이지에서 데이터 추출 |
| `browser.click` | 버튼/링크 클릭 등 DOM 조작 |
| `browser.input` | 폼 입력, 검색어 입력 등 |
| `browser.wait` | 특정 요소 로딩 대기 |
| `browser.cookies` | 쿠키/세션 관리 (로그인 유지) |

> **구현**: Rust `chromiumoxide` 또는 Playwright (자식 프로세스) 사용.
> Confluence, Notion, Jira 등 로그인 필요한 SaaS에 접근 가능.

| `data.summarize` | 데이터 요약 통계 |

#### 4-5. 시스템 도구

|------|------|
| `system.info` | 시스템 정보 조회 |
