# Recursive Memory Consolidation - Consensus Plan (v4)

> Generated by /cplan (Claude + Codex consensus)
> Date: 2026-03-19

## Requirement

기억 시스템을 recursive consolidation으로 재설계:
- m(n+1) = F(m(n), d(n), notes) — 대화 종료 시 기억 통합
- 불변 대화 다이제스트 + 가변 통합 기억 2-layer 구조
- 구조화된 섹션별 기억 관리

## Design Decisions (Codex 합의)

1. **요약 vs 통합**: Hybrid (C) — 대화 중 summary 유지, 대화 끝에 대화 스냅샷으로 digest 생성
2. **Raw 노트 보존**: Archive (C) — 통합 후 archive로 이동, 재빌드 가능
3. **공유 노트**: Exclude (C) — 공유 노트는 통합하지 않고 별도 채널 유지

## Agreed Plan

### Architecture

```
대화 중:     memory_note 도구로 개별 노트 캡처 (기존대로)
             summary 생성 (기존 maybeGenerateSummary, 대화 내 연속성용)

대화 전환 시: d(n) = digest(conversation_messages + captured_notes)
                     ↑ 대화 스냅샷에서 직접 생성 (summary 의존 X)
             m(n+1) = F(m(n), d(n))
             raw notes → archive
             m(n+1) → 다음 대화의 [CONSOLIDATED MEMORY] 프롬프트 섹션
```

### Storage (시스템 메모리 전용 영역)

일반 vault 노트와 분리된 시스템 메모리 영역:

```
app_data_dir()/memory/<agent_id>/
  ├── consolidated.md          # m(n) 현재 통합 기억
  ├── snapshots/               # m(n-1), m(n-2)... 이전 스냅샷 (rollback용)
  │   └── v{N}_{timestamp}.md
  └── digests/                 # d(0), d(1)... 불변 대화 다이제스트
      └── {conversation_id}.md
```

일반 vault (기존):
```
~/.window-agent/vault/agents/<agent_id>/
  ├── knowledge/*.md           # 활성 raw 노트
  └── archive/*.md             # 통합 완료된 raw 노트
```

### Note Provenance (신규)

memory_note 도구로 생성되는 vault 노트에 `source_conversation` 필드 추가:

```yaml
---
id: <uuid>
agent: <agent_id>
type: knowledge
source_conversation: <conversation_id>   # 신규: 어떤 대화에서 생성되었는지
tags: [...]
confidence: 0.8
---
```

### Consolidation Checkpoint (DB)

conversations 테이블에 consolidation 상태 추가:

```sql
ALTER TABLE conversations ADD COLUMN digest_id TEXT;         -- 생성된 digest ID (null = 미처리)
ALTER TABLE conversations ADD COLUMN consolidated_at TEXT;    -- consolidation 완료 시각
```

에이전트 레벨:
```sql
-- agents 테이블에 추가하지 않고, consolidated.md의 frontmatter version으로 관리
```

### Consolidated Memory Schema (m(n) 구조)

```markdown
---
agent: <agent_id>
version: <n>
last_consolidated_at: <timestamp>
last_digest_conversation: <conversation_id>
---

## Profile
- 사용자는 Tauri+React 풀스택 개발자 [c0, c3]

## Preferences
- 병렬 작업 시 TeamCreate 선호 [c2]

## Active Projects
- 기억 시스템 재설계 진행 중 [c9]

## Rules
- PR은 Codex 리뷰 8.0 이상 필요 [c2, c5]

## Recent
- 최근 recursive memory 설계 논의 [c9]
```

### Digest 생성 입력

summary에 의존하지 않고 대화 스냅샷에서 직접 생성:
- 대화 메시지 (최근 N개 또는 토큰 예산 내)
- 해당 대화의 tool_call_logs에서 memory_note create/update 내역
- 해당 대화의 captured notes (source_conversation으로 필터)

### Trigger & Idempotency

- **트리거**: 대화 전환 시 (selectConversation, openAgentChat, createNewConversation 등)
- **조건**: 이전 대화의 `digest_id`가 null이고 메시지가 1개 이상
- **직렬화**: 에이전트당 한 번에 하나의 consolidation만 실행 (mutex/flag)
- **실패 복구**: 앱 시작 시 `digest_id IS NULL AND messages > 0`인 대화 감지 → 재시도
- **짧은 대화**: 메시지 3개 미만이면 consolidation 건너뜀 (의미 있는 내용 없음)

### Model Selection

- digest/consolidation은 **저렴한 모델** 사용 (구조화된 작업이므로)
- 기본: 에이전트 모델과 동일, 설정에서 override 가능
- 온도: 0 (결정론적 출력)

## Implementation Steps

### 1. Note provenance + DB checkpoint {parallel: false}
- Actions:
  - vault/note.rs Frontmatter에 `source_conversation: Option<String>` 추가
  - tool_commands.rs memory_note create에서 conversation_id를 source_conversation으로 자동 주입
  - DB migration: conversations에 digest_id, consolidated_at 컬럼 추가 (idempotent ALTER TABLE)
  - models.rs ConversationDetail에 digest_id, consolidated_at 추가 (#[serde(default)])
  - operations.rs에 update_conversation_consolidation_status 함수 추가
- Files: vault/note.rs, tool_commands.rs, migrations.rs, models.rs, operations.rs, chat_commands.rs
- Dependencies: none

### 2. Vault archive 인프라 {parallel: true}
- Actions:
  - vault/mod.rs에 archive_note(id, agent_id) 함수 추가 — knowledge/ → archive/ 이동
  - archive/ 디렉토리 생성 (없으면)
  - list_notes()에서 archive/ 경로 제외
  - search()에서 archive/ 경로 제외
  - recall()에서 archive/ 경로 제외
- Files: vault/mod.rs, vault/search.rs
- Dependencies: none

### 3. 시스템 메모리 저장소 {parallel: true}
- Actions:
  - Rust에 SystemMemoryManager 구조체 추가 (consolidated.md, digests/, snapshots/ 관리)
  - read_consolidated(agent_id) → Option<String> (내용 반환)
  - write_consolidated(agent_id, content, version) → 이전 버전 스냅샷 자동 보존
  - write_digest(agent_id, conversation_id, content) → digests/ 저장
  - list_digests(agent_id) → Vec<DigestMeta>
  - 경로: app_data_dir()/memory/<agent_id>/
  - Tauri command: read_consolidated_memory(agent_id), get_system_memory_path()
- Files: src-tauri/src/memory/ (신규 모듈), src-tauri/src/commands/chat_commands.rs
- Dependencies: none

### 4. Digest 생성 엔진 {parallel: false}
- Actions:
  - Tauri command: generate_digest(conversation_id)
  - 입력 수집: DB에서 conversation messages + tool_call_logs 중 memory_note 관련 + vault에서 source_conversation 매칭 노트
  - LLM 호출: 메시지 + 노트 → 구조화된 다이제스트 (온도 0, 저렴한 모델)
  - SystemMemoryManager.write_digest()로 저장
  - DB: conversation.digest_id 업데이트
  - 해당 대화의 raw notes를 archive로 이동
- Files: src-tauri/src/commands/chat_commands.rs (또는 memory_commands.rs 신규)
- Dependencies: Step 1, Step 2, Step 3

### 5. Consolidation 함수 F {parallel: false}
- Actions:
  - Tauri command: consolidate_memory(agent_id)
  - 현재 m(n) 읽기 (SystemMemoryManager.read_consolidated)
  - 최신 미통합 digests 읽기
  - LLM 호출: m(n) + d(n) → m(n+1) (구조화된 마크다운, 온도 0)
  - SystemMemoryManager.write_consolidated()로 저장 (스냅샷 자동)
  - DB: conversation.consolidated_at 업데이트
- Files: src-tauri/src/commands/chat_commands.rs
- Dependencies: Step 3, Step 4

### 6. Frontend 통합 트리거 {parallel: false}
- Actions:
  - conversationStore에 consolidation 트리거 로직 추가
  - 트리거 시점: 대화 전환 시 이전 대화의 digest_id가 null이고 메시지 있을 때
  - 에이전트당 mutex (진행 중이면 스킵)
  - 비동기 실행 (fire-and-forget, 대화 전환 차단 안 함)
  - 앱 시작 시: Tauri command list_pending_consolidations() → digest_id IS NULL AND message_count >= 3 조회 → 재시도
  - tauriCommands.ts에 generateDigest, consolidateMemory, readConsolidatedMemory, listPendingConsolidations 래퍼 추가
  - types.ts: ConversationDetail에 digest_id?, consolidated_at? 추가
  - conversationStore: consolidatedMemory를 resetChatContext, clearAgentChat, startNewAgentConversation에서 null로 리셋
- Files: conversationStore.ts, chatCommands.ts, tauriCommands.ts, types.ts, resetHelper.ts
- Dependencies: Step 4, Step 5

### 7. 프롬프트 주입 변경 {parallel: false}
- Actions:
  - conversationStore에 `consolidatedMemory: string | null` store 필드 추가
  - 에이전트/대화 전환 시 readConsolidatedMemory(agentId) 호출 → store에 로드
    - selectConversation(): detail.agent_id로 로드
    - openAgentChat(): agentId로 로드
    - prepareForAgent(): agentId로 로드
  - buildConversationContext()에 `consolidatedMemory?: string` 파라미터 추가
  - chatFlowStore streamOneTurn + regenerateStream:
    - `conversationStore.getState().consolidatedMemory`를 buildConversationContext에 전달
  - buildConversationContext 내부:
    - consolidatedMemory가 있으면 [CONSOLIDATED MEMORY] 섹션으로 주입
    - 없으면 기존 방식(raw notes) fallback
    - 배움모드: 통합 기억 + 현재 대화 raw notes 모두 (예산 1500)
    - 일반 모드: 통합 기억만 (예산 800)
  - draft/no-conversation 상태: selectedAgentId로 로드
  - memory_note recall: 별도 retrieval path 유지 (consolidated와 분리)
- Files: chatHelpers.ts, chatFlowStore.ts, conversationStore.ts
- Dependencies: Step 3, Step 6

### 8. Consolidation 프롬프트 (i18n) {parallel: true}
- Actions:
  - digest 생성용 프롬프트 (ko/en): 대화에서 기억할 가치 있는 정보 추출
  - consolidation F용 프롬프트 (ko/en): m(n) + d(n) → m(n+1) 병합
  - 출력 스키마: Profile, Preferences, Active Projects, Rules, Recent
  - 각 항목에 source conversation ID 참조 필수 [cN] 형식
  - 충돌 해결 규칙: 최신 정보가 우선, 확인된 정보는 유지
- Files: ko/prompts.json, en/prompts.json
- Dependencies: none

### 9. Export/Import 호환 {parallel: false}
- Actions:
  - export_commands.rs: conversation INSERT에 digest_id, consolidated_at 추가
  - models.rs: ConversationDetail에 #[serde(default)]로 역호환
  - Export 정책: digests + archived raw notes 포함 (rebuild 가능), consolidated.md는 제외 (rebuildable cache)
  - export_agent()에서 app_data_dir()/memory/<agent_id>/digests/ 를 ZIP에 포함
  - import_agent()에서:
    - 대화 ID 리맵핑 시 digest 파일명도 함께 리맵핑 ({old_conv_id}.md → {new_conv_id}.md)
    - digest frontmatter 내 conversation 참조도 리맵핑
    - 임포트된 대화의 digest_id, consolidated_at을 **null로 리셋** (import 후 첫 전환 시 재통합)
    - archived raw notes의 source_conversation도 새 ID로 리맵핑
    - consolidated.md 미생성 (자동 rebuild)
  - 이전 버전 export (digest_id 없음) → serde(default)로 null 역직렬화
- Files: export_commands.rs, models.rs, operations.rs
- Dependencies: Step 1, Step 3

### 10. 테스트 {parallel: true}
- Actions:
  - Rust: archive_note 이동 + list/search/recall 제외 테스트
  - Rust: SystemMemoryManager CRUD + 스냅샷 보존 테스트
  - Rust: source_conversation frontmatter 직렬화/역직렬화 테스트
  - Rust: DB consolidation checkpoint CRUD 테스트
  - Rust: export/import 역호환 (digest_id 없는 이전 export → null 기본값)
  - Frontend: buildConversationContext가 consolidated 사용하는지 테스트
- Files: Rust 테스트, frontend 테스트
- Dependencies: Step 1-9

## Acceptance Criteria

- [ ] memory_note 생성 시 source_conversation 자동 주입
- [ ] conversations에 digest_id, consolidated_at checkpoint 필드
- [ ] 대화 전환 시 비동기로 digest + consolidation 실행
- [ ] digest는 대화 스냅샷(메시지+노트)에서 직접 생성 (summary 의존 X)
- [ ] m(n+1) = F(m(n), d(n)) 구조로 통합 기억 생성
- [ ] 통합 기억 구조화 섹션 (Profile, Preferences, Projects, Rules, Recent)
- [ ] 각 항목에 source conversation 참조 [cN]
- [ ] 시스템 메모리 전용 영역 (app_data_dir()/memory/) — vault과 분리
- [ ] raw notes가 통합 후 archive로 이동
- [ ] archived notes가 search/recall/list에서 제외
- [ ] consolidated memory를 store에 로드 → buildConversationContext에 전달 (전 경로)
- [ ] 프롬프트에 [CONSOLIDATED MEMORY] 주입 (fallback: 기존 raw notes)
- [ ] 배움모드에서 통합 기억 + raw notes 모두 주입
- [ ] import에서 digest_id/consolidated_at NULL 리셋 (자동 재빌드)
- [deferred] export에 digests 포함 + import 시 ID 리맵핑 → v2
- [ ] 대화 전환이 consolidation에 의해 차단되지 않음
- [ ] 에이전트당 consolidation 직렬화 (중복 실행 방지)
- [ ] 짧은 대화 (메시지 <3) 건너뜀
- [ ] 앱 시작 시 미처리 대화 감지 → 재시도
- [ ] digest 불변 보존 + m(n) 스냅샷 보존 (rollback/rebuild 가능)
- [ ] 한/영 프롬프트 완비
- [ ] 테스트 통과
