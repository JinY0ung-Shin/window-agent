# P2P Direct Internet Connection - Consensus Plan

> Generated by /cplan (Claude + Codex consensus)
> Consensus reached: Round 4
> Date: 2026-03-18

## Requirement

인터넷을 통한 직접 P2P 연결 지원 (Multiaddr 기반). 사용자가 외부 IP:port를 초대 코드에 포함시켜 인터넷 상의 피어와 직접 연결할 수 있도록 한다. 설정 가능한 고정 리슨 포트를 제공하여 포트포워딩과 호환되도록 한다.

## Context

- 현재 mDNS(LAN) 전용. libp2p 0.54, TCP+Noise+Yamux 스택.
- ContactCard에 addresses/relay_hints 필드 있지만 빈 배열로 생성됨.
- P2PManager에 dial(peer_id, addrs) 메서드 이미 존재.
- Swarm은 백그라운드 tokio task 소유, command channel로 통신.
- SettingsModal.tsx에 P2P 설정 UI 존재 (network_enabled 토글).
- DB는 rusqlite, 마이그레이션은 db.rs에서 수행.

## Agreed Plan

### Architecture
- 기존 libp2p 인프라를 그대로 활용. 초대 코드에 Multiaddr 주소를 넣고, 수락 시 해당 주소로 자동 dial + 주소를 contact에 별도 저장.
- 추후 relay/DCUtR 확장에 영향 없음.
- Address Format: 정규화된 Multiaddr만 허용 (e.g. `/ip4/1.2.3.4/tcp/4001`, `/ip6/2001:db8::1/tcp/4001`). `/p2p/<peer_id>` 접미사는 자동 strip.
- Port Rules: `listen_port = None` means random(ephemeral). `listen_port = 0` is NOT valid. Valid range: 1-65535.

### Key Decisions
- **Duplicate peer_id**: 이미 알려진 peer_id의 새 초대 수락 시 기존 contact의 addresses_json, invite_card_raw, agent_name, agent_description을 새 초대 값으로 업데이트. local_agent_id는 사용자가 AcceptTab에서 선택한 값이 있으면 업데이트, 없으면 유지.
- **Migration**: 기존 마이그레이션 프레임워크의 새 번호 마이그레이션으로 구현.
- **TypeScript types**: `addresses_json: string | null` (nullable, not optional).
- **Reconnect timeout UX**: "아직 연결되지 않았습니다" (soft, not hard failure).

## Implementation Steps

### 0. [Backend: 설정 가능한 고정 리슨 포트] {parallel: true}
- p2p-settings.json store에 `listen_port: Option<u16>` 설정 추가
- manager.rs start()에서 설정 읽어 listen address 결정
- p2p_commands.rs에 `p2p_get_listen_port` / `p2p_set_listen_port` 커맨드 추가 (port=0 거부)
- SettingsModal.tsx에 리슨 포트 설정 섹션: configured/active 포트 표시, 재시작 안내
- **Files**: manager.rs, p2p_commands.rs, p2pCommands.ts, SettingsModal.tsx
- **Dependencies**: 없음

### 1. [Backend: 리슨 주소 공유 스냅샷 + connection info API] {parallel: true}
- P2PManager에 `listen_addresses: Arc<Mutex<Vec<String>>>` 필드 추가
- SwarmEvent::NewListenAddr → 스냅샷 업데이트 (loopback + unspecified 필터링)
- SwarmEvent::ListenerClosed → 스냅샷에서 제거, stop() 시 clear
- `p2p_get_connection_info` → `{ peer_id, configured_listen_port, active_listen_port, listen_addresses, status }`
- 네트워크 비활성 시 active_listen_port = None, listen_addresses = []
- **Files**: manager.rs, p2p_commands.rs, p2pCommands.ts
- **Dependencies**: 없음

### 2. [Backend: 초대 생성에 주소 포함 + Multiaddr 검증] {parallel: true}
- `p2p_generate_invite`에 `addresses: Vec<String>` 파라미터 추가
- Multiaddr::try_from()으로 검증, /p2p/ strip
- networkStore.ts의 generateInvite에 addresses 추가
- **Files**: p2p_commands.rs, p2pCommands.ts, networkStore.ts
- **Dependencies**: 없음

### 3. [Backend: DB 마이그레이션 + 초대 수락 시 주소 저장 + 자동 dial] {parallel: true}
- **Migration**: 새 번호 마이그레이션으로 `ALTER TABLE contacts ADD COLUMN addresses_json TEXT` (idempotent)
- contacts CRUD 쿼리 전부 addresses_json 포함으로 업데이트
- ContactRow, ContactUpdate에 `addresses_json: Option<String>` 추가
- p2p_accept_invite async 변환, 실행 순서:
  1. ContactCard 파싱/검증
  2. addresses → addresses_json 직렬화
  3. DB 저장 (중복 peer_id 시 기존 contact 업데이트: addresses_json, invite_card_raw, agent_name, agent_description, 선택된 local_agent_id)
  4. add_known_peer()
  5. best-effort dial
- **Files**: db.rs, p2p_commands.rs
- **Dependencies**: 없음

### 4. [Backend: 수동 dial 커맨드 (재연결)] {parallel: true}
- `p2p_dial_peer(contact_id)` → addresses_json에서 주소 추출 (만료 무관)
- 주소 없으면 에러, 있으면 manager.dial() (fire-and-forget)
- **Files**: p2p_commands.rs, p2pCommands.ts
- **Dependencies**: 없음

### 5. [Frontend: InviteDialog 주소 입력 UI] {parallel: false}
- GenerateTab:
  - p2p_get_connection_info로 LAN 주소 목록 (체크박스)
  - Raw Multiaddr 수동 입력 텍스트 영역 (IPv4/IPv6)
  - IP만 입력 시 active_listen_port 자동 적용 (편의, 선택적)
  - 네트워크 비활성 시에도 수동 Multiaddr 입력은 가능
- AcceptTab:
  - URL-safe no-padding base64 디코드 헬퍼로 미리보기 (unverified 라벨)
  - 주소 포함 여부 표시
- **Files**: InviteDialog.tsx, networkStore.ts
- **Dependencies**: Step 0, 1, 2

### 6. [Frontend: 연락처 재연결 + TypeScript 타입] {parallel: false}
- ContactRow 타입에 `addresses_json: string | null` 추가
- ContactDetail에 재연결 버튼 (addresses_json 없으면 비활성화 + 툴팁)
- 클릭 → p2p_dial_peer → "연결 시도 중..." → p2p:peer-connected 이벤트로 성공 확인
- 10초 후 연결 없으면 "아직 연결되지 않았습니다" (soft message)
- **Files**: ContactDetail.tsx, p2pCommands.ts, networkStore.ts
- **Dependencies**: Step 3, 4

### 7. [테스트] {parallel: false}
- cargo test (기존 P2P 테스트 통과 확인)
- DB 마이그레이션 테스트 (기존 DB 업그레이드 + contact listing)
- Multiaddr 검증 테스트 (유효/무효, /p2p/ strip)
- addresses_json 저장/조회 라운드트립
- duplicate peer_id 수락 시 주소 업데이트
- p2p_dial_peer 주소 없는 contact 에러
- npm test
- **Files**: db.rs, invite.rs, contact_card.rs, p2p_commands.rs
- **Dependencies**: Step 0-6

## Acceptance Criteria

- [ ] 고정 리슨 포트 설정 가능 (Settings UI, 0은 거부, 재시작 필요)
- [ ] configured vs active 포트 구분 표시
- [ ] 초대 코드에 Multiaddr 주소 포함
- [ ] 주소 입력 시 Multiaddr 검증, IPv4/IPv6 지원
- [ ] 초대 수락 시 주소를 addresses_json에 저장 + 자동 연결 시도
- [ ] 기존 peer_id 재초대 시 주소 업데이트 (중복 생성 방지)
- [ ] 수동 재연결 — 저장된 주소로 dial (만료 무관)
- [ ] 주소 없는 연락처는 재연결 비활성화 + 안내
- [ ] AcceptTab에서 초대 코드 미리보기 (unverified, URL-safe base64 디코드)
- [ ] 기존 mDNS(LAN) 연결 유지
- [ ] DB 마이그레이션 안전 (기존 DB 업그레이드 + 테스트)
- [ ] 모든 기존 테스트 통과 + 새 테스트 추가

## Codex Final Assessment

- **Strengths**:
  - 기존 libp2p 아키텍처를 재사용하여 병렬 연결 경로 도입 없이 깔끔한 확장
  - reconnect 로직이 invite 만료와 분리되어 정확한 설계
  - frontend/backend 계약이 잘 설계됨: runtime info, Multiaddr 검증, unverified preview 분리
  - best-effort dial 시퀀싱이 기존 mDNS 경로를 fallback으로 보존
- **Overall**: 승인 가능. 남은 이슈는 구현 단계의 사소한 명확화 수준.
