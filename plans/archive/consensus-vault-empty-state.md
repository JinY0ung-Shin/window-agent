# Vault Empty State Fix - Consensus Plan

> Generated by /cplan (Claude + Codex consensus)
> Consensus reached: Round 2
> Date: 2026-03-18

## Requirement

Vault 창에 저장된 노트가 없을 때 무한 로딩 스켈레톤 대신 빈 상태 UI 표시

## Agreed Plan

vaultStore에 `notesStatus` enum (`idle | loading | loaded | error`)과 `_loadVersion` 카운터를 추가하여:
- 로딩 중 vs 비어있음 vs 에러 상태를 구분
- 에이전트 전환 시 stale notes 방지 (notes: [] 초기화)
- 비동기 경쟁 조건 방지 (load version guard)

## Implementation Steps

1. vaultStore에 notesStatus와 loadVersion 추가 {parallel: false}
   - Actions:
     a. VaultState 인터페이스에 `notesStatus: 'idle' | 'loading' | 'loaded' | 'error'` 추가
     b. store 외부에 `let _loadVersion = 0` 카운터 추가
     c. `loadNotes` 수정:
        - 시작: `_loadVersion++`, version 캡처, `set({ notes: [], notesStatus: 'loading' })`
        - 성공: version 일치 시만 `set({ notes, notesStatus: 'loaded' })`
        - 실패: version 일치 시만 `set({ notes: [], notesStatus: 'error' })`
     d. `clear()` 시: `_loadVersion++`, `notesStatus: 'idle'` 리셋
   - Files: src/stores/vaultStore.ts

2. NoteListPane에서 notesStatus 사용 {parallel: false}
   - Actions:
     a. store에서 `notesStatus` 구독
     b. 렌더링 분기:
        - `idle` 또는 `loading` → 스켈레톤
        - `error` → 에러 메시지
        - `loaded` + `displayNotes.length === 0` → 빈 상태 메시지
        - `loaded` + `displayNotes.length > 0` → 노트 목록
   - Files: src/components/vault/NoteListPane.tsx
   - Dependencies: Step 1

## Acceptance Criteria

- [ ] Vault에 노트가 0개일 때 "노트가 없습니다" 빈 상태가 표시됨
- [ ] 초기 로딩 중에는 스켈레톤 애니메이션이 표시됨
- [ ] 에이전트 전환 시 스켈레톤 → 결과 표시가 정상 동작 (stale notes 없음)
- [ ] 빠른 에이전트 전환 시 이전 요청 결과가 무시됨 (race condition 방지)
- [ ] 로드 에러 시 에러 메시지 표시
- [ ] TypeScript 타입 체크 통과
