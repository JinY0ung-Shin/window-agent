# Learning Mode (배움모드) - Consensus Plan

> Generated by /cplan (Claude + Codex consensus)
> Consensus reached: Round 4
> Date: 2026-03-19

## Requirement

에이전트에게 무언가를 가르칠 때, 에이전트가 학생의 자세로 빠릿빠릿하게 기억을 사용하고 효과적으로 기억하는 "배움모드" 추가

## Agreed Plan

대화별로 토글 가능한 배움모드를 추가하여, 활성화 시 에이전트가 학생처럼 적극적으로 기억을 생성하고, 배운 내용을 확인하고, 기존 기억과 연결하는 행동을 하도록 한다.

### Architecture

- **대화별 DB 컬럼** + 시스템 프롬프트 주입 방식
- SCHEMA_VERSION은 유지하고 **idempotent ALTER TABLE**로 증분 마이그레이션
- **conversationStore**에 모든 학습모드 상태 집중 (단일 소유자)
- `getCurrentLearningMode()` 셀렉터로 단일 진실 소스
- memory_note는 이미 auto tier → 별도 승격 불필요

## Implementation Steps

### 1. DB 스키마 + Rust 백엔드 {parallel: false}
- **CREATE TABLE** conversations에 `learning_mode INTEGER DEFAULT 0` 추가
- **SCHEMA_VERSION 변경하지 않음** (기존 DB 삭제 방지)
- `ensure_schema()` 후 `run_incremental_migrations()` 추가:
  - `PRAGMA table_info(conversations)`로 `learning_mode` 컬럼 존재 여부 확인
  - 없으면 `ALTER TABLE conversations ADD COLUMN learning_mode INTEGER DEFAULT 0` 실행
  - 이 패턴은 향후 다른 컬럼 추가에도 재사용 가능
- `models.rs` ConversationDetail에 `learning_mode: bool` 추가 (`#[serde(default)]`)
- `operations.rs`: `set_learning_mode(id, enabled)`, SELECT에서 learning_mode 포함
- Tauri command: `set_learning_mode(conversation_id, enabled: bool)`
- **Files**: `migrations.rs`, `schema.rs`, `models.rs`, `operations.rs`, `chat_commands.rs`
- **Dependencies**: none

### 2. Export/Import 업데이트 {parallel: true}
- `export_commands.rs` line 377-388의 INSERT 문에 `learning_mode` 컬럼 추가
- `conv_detail.learning_mode`를 파라미터에 포함 (false fallback)
- 이전 버전 export 파일에서 learning_mode가 없어도 `#[serde(default)]`로 false 역직렬화
- ConversationExport는 이미 ConversationDetail을 serde 직렬화하므로 export는 자동 포함
- **Files**: `export_commands.rs`
- **Dependencies**: Step 1

### 3. 프론트엔드 상태 관리 {parallel: false}
- ALL learning-mode state lives in **conversationStore** (single owner):
  - `currentLearningMode: boolean` (synced from DB for active conversation)
  - `draftLearningMode: boolean` (pre-conversation draft state)
  - `getCurrentLearningMode(): boolean` selector
    → `currentConversationId ? currentLearningMode : draftLearningMode`
  - `toggleLearningMode()`:
    → if currentConversationId: DB save + update currentLearningMode
    → else: toggle draftLearningMode
  - `resetLearningModeState()`: reset both to false (called from resetHelper)
- **Sync points** (matching existing selectConversation pattern):
  - `selectConversation(id)`: fetches detail → set `currentLearningMode = detail.learning_mode ?? false`, reset draft
  - On conversation creation (in `chatFlowStore.sendNormalMessage`):
    → `cmds.setLearningMode(newConvId, draftLearningMode)` if draft is true
    → set `currentLearningMode = draftLearningMode`, reset draft
  - `resetChatContext()`/`createNewConversation()`: call `resetLearningModeState()`
  - `openAgentChat()` no-conv branch: reset both
  - `startNewAgentConversation()`: reset both
  - `clearAgentChat()`: if wasActive, reset both
- chatFlowStore only READS via `conversationStore.getCurrentLearningMode()`
- ConversationListItem is NOT widened
- Tauri command wrapper in `chatCommands.ts`, re-exported via `tauriCommands.ts`
- **Files**: `types.ts`, `chatCommands.ts`, `tauriCommands.ts`, `conversationStore.ts`, `chatFlowStore.ts`, `resetHelper.ts`
- **Dependencies**: Step 1

### 4. 배움모드 시스템 프롬프트 작성 {parallel: true}
- i18n prompts 네임스페이스에 learningMode 섹션:
  - a. **학생의 자세** — 배운 내용을 자신의 말로 요약하여 확인
  - b. **검색 우선** — memory_note search → 기존 노트 있으면 update, 없으면 create
  - c. **연결** — 새 정보를 기존 기억과 연결, related_ids 활용
  - d. **질문** — 불명확한 부분은 확인 질문으로 이해도 검증
  - e. **구조화** — 카테고리 태그 활용, 한 턴에 최대 3개 기억 생성 제한
  - f. **기존 기억 참조** — 답변 시 관련 기억을 인용하여 학습 누적 표현
- **Files**: `ko/prompts.json`, `en/prompts.json`
- **Dependencies**: none

### 5. 기억 토큰 예산 동적 조정 {parallel: true}
- `buildMemorySection(notes, maxTokens=500)`에 파라미터 추가
- `buildPromptReadySlice(notes, maxTokens=500)`에도 동일
- `buildVaultMemorySection(notes, maxTokens?)` → 전달
- `buildConversationContext`에 `learningMode` 파라미터 추가
- `learningMode=true`이면 `maxTokens=1500`
- **Files**: `chatHelpers.ts`, `memoryAdapter.ts`
- **Dependencies**: none

### 6. 시스템 프롬프트 주입 + 전체 경로 통합 {parallel: false}
- `buildConversationContext()`에서 learningMode일 때:
  - (a) `[LEARNING MODE]` 프롬프트 섹션 삽입 (스킬 뒤, 메모리 앞)
  - (b) 기억 예산 1500으로 증가
- `streamOneTurn()`: `conversationStore.getCurrentLearningMode()` → `buildConversationContext`에 전달
- `regenerateStream()`: 동일하게 learningMode 전달
- memory_note가 disabled/deny인 에이전트의 경우 토글 시 UI 경고
- **Files**: `chatHelpers.ts`, `chatFlowStore.ts`
- **Dependencies**: Step 3, Step 4, Step 5

### 7. 토글 UI + UX 안내 {parallel: false}
- ChatInput 영역에 `GraduationCap` 토글 버튼
- 활성화 시 버튼 색상 변경 + 인디케이터
- 첫 활성화 시 안내 툴팁: "배움모드에서 저장된 기억은 이 에이전트의 모든 대화에 적용됩니다"

### 8. 테스트 {parallel: true}
- **Rust**: learning_mode DB CRUD + idempotent migration (ALTER TABLE 2회 실행해도 안전)

- **Files**: Rust db 테스트, frontend 서비스 테스트

## Codex Final Assessment

- **Strengths**:
  - conversationStore 단일 소유자 패턴이 기존 selectConversation 흐름과 잘 맞음
  - ConversationListItem을 확장하지 않고 currentLearningMode를 별도 관리하는 방식이 합리적
  - chatFlowStore가 getCurrentLearningMode()만 읽는 구조로 중복 로직 제거
- **Overall**: 상태 소유권 문제가 해결되어 구현 가능한 수준
- **Minor note**: resetHelper.ts에서 resetLearningModeState() 호출 추가 필요
