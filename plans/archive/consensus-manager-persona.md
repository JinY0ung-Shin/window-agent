# Manager Persona Enhancement - Consensus Plan

> Generated by /cplan (Claude + Codex consensus)
> Consensus reached: Round 3
> Date: 2026-03-18

## Requirement

팀장 에이전트의 페르소나를 강화:
1. 조직의 한 팀을 담당하는 팀장이라는 정체성 확립
2. 시스템(window-agent)에 대한 self-awareness 부여

## Agreed Plan

### Architecture
- **정적 페르소나 파일**: 리더십 스타일, 역할 정체성, 팀원 관리 규칙
- **런타임 동적 주입**: 회사명({{company_name}}), 시스템 능력([SYSTEM CONTEXT])
- **기존 사용자 안전 갱신**: startup helper로 파일별 정확 비교 후 갱신

## Implementation Steps

### 1. IDENTITY.md 개선 — 조직 팀장 정체성 강화 {parallel: true}
- Actions: '범용 AI 도우미' → '회사의 AI 팀을 이끄는 팀장'으로 정체성 재정의
- {{company_name}} 플레이스홀더 도입 (런타임 치환)
- 시스템 기능은 여기에 넣지 않음 (Step 4에서 런타임 주입)
- Files: `src-tauri/resources/default-agent/IDENTITY.md`
- Dependencies: none

### 2. SOUL.md 개선 — 팀장다운 가치관과 행동 패턴 {parallel: true}
- Actions: 팀장으로서의 리더십, 팀원 관리 관점 강화
- 기존 '직원 안내' → '팀원 관리 및 위임' 톤으로 조정
- 자신의 한계 인지는 유지하되, 구체적 기능 나열은 하지 않음
- Files: `src-tauri/resources/default-agent/SOUL.md`
- Dependencies: none

### 3. AGENTS.md 개선 — 팀원 관리 규칙 강화 {parallel: true}
- Actions: '직원 안내' → '팀원 업무 위임 및 협업' 관점으로 재작성
- 팀장이 팀원의 역할을 파악하고 적재적소에 배치하는 느낌
- Files: `src-tauri/resources/default-agent/AGENTS.md`
- Dependencies: none

### 4. assembleManagerPrompt 확장 — 회사명 + [SYSTEM CONTEXT] 동적 주입 {parallel: true}
- Actions:
  - assembleManagerPrompt에 companyName + enabledToolNames 파라미터 추가
  - 프롬프트 내 {{company_name}}을 실제 값으로 치환 (빈 값 → '우리 회사' fallback)
  - [SYSTEM CONTEXT] 섹션: 활성화된 도구 이름만 간결하게 나열
  - chatFlowStore에서 도구 로드 완료 후 프롬프트 조립하는 순서 보장
  - 모든 assembleManagerPrompt 호출부 업데이트
- Files: `src/services/personaService.ts`, `src/stores/chatFlowStore.ts`
- Dependencies: none

### 5. 기존 사용자 페르소나 갱신 — startup helper {parallel: false}
- Actions:
  - Rust에 `refresh_default_manager_persona(app_handle)` 함수 추가
  - **매 startup마다 호출** (seed 후뿐 아니라 기존 설치에서도)
  - 파일별 독립 비교: 각 .md 파일을 이전 번들 기본값과 정확히 비교
  - **비교 시 line ending normalize + trailing newline trim** 적용
  - old default와 일치 → 새 기본값으로 교체
  - 다르면(사용자 수정) → 해당 파일 보존
  - USER.md는 항상 보존
  - old default 문자열은 named constants로 관리
- Files: `src-tauri/src/commands/agent_commands.rs`
- Dependencies: Step 1, 2, 3

### 6. 테스트 업데이트 {parallel: false}
- Actions:
  - personaService.test.ts: companyName 치환, 빈 값 fallback, [SYSTEM CONTEXT] 주입
  - assembleManagerPrompt 기존 테스트를 새 시그니처에 맞게 업데이트
  - store 레벨 테스트/mock도 확인
  - Rust: refresh helper 테스트 — 미수정 파일 갱신, 수정된 파일 보존, 부분 수정 케이스
- Files: `src/services/__tests__/personaService.test.ts`, `src-tauri/src/commands/agent_commands.rs` (tests)
- Dependencies: Step 4, 5

## Acceptance Criteria

- [ ] 팀장이 '나는 {{회사명}}의 팀장이다'라는 정체성을 인식
- [ ] 회사명이 시스템 프롬프트에 동적 주입됨 (빈 값 시 '우리 회사' fallback)
- [ ] [SYSTEM CONTEXT]에 현재 활성 도구 목록이 간결하게 포함됨
- [ ] 기존 사용자의 미수정 페르소나가 파일별로 안전하게 갱신됨
- [ ] 사용자가 수정한 파일은 절대 덮어쓰지 않음
- [ ] personaService 테스트 + refresh helper 테스트 통과
- [ ] 기존 테스트 전체 통과
