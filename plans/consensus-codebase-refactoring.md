# Codebase Refactoring — Consensus Plan

> Generated by /cplan (Claude + Codex consensus)
> Consensus reached: Round 3
> Date: 2026-03-19

## Requirement

전체 코드베이스 리팩토링 — 22건의 이슈를 우선순위와 의존성 기반으로 실행 순서 확정.

## Agreed Plan

3개 독립 Track으로 분리. Track 내 순차 실행, Track 간 병렬 실행. 매 슬라이스 후 build+test 게이트 필수. Gap-closure 슬라이스로 acceptance criteria 100% 달성.

### Track 구조 (disjoint write scope)
- **Track A**: Backend 안정성 & 구조 (Rust only)
- **Track B**: Frontend Chat/Team & 스토어 (TS/TSX only)
- **Track C**: UI/CSS 정리 & 아카이브 (CSS, settings components, plans — Rust 미수정)

### Verification
매 슬라이스 완료 시: `cargo test` + `npm test` + 대상 기능 수동 smoke test (DM chat, team run, vault, network, settings)

## Implementation Steps

### Track A: Backend 안정성 & 구조

#### A-1. Production panic path 제거 {parallel: false}
- **Actions**:
  - p2p/manager.rs: Mutex `.lock().unwrap()` → `.lock().map_err()?` 또는 `.expect("context")`
  - api.rs: 파싱/HTTP unwrap → Result 전파
  - lib.rs: startup expect() 검토 — graceful fallback 또는 명확 메시지
  - vault/mod.rs: 파일 I/O unwrap → Result 전파
  - tool_commands.rs, vault/links.rs, browser.rs 등 **모든 production 파일**의 bare unwrap 제거
  - 테스트 코드 unwrap은 제외
- **Implementation Policy** (Codex OBJ-3-1 반영):
  - `expect("context")` — startup invariant 또는 의도적 fatal 가정에만 사용
  - mutex lock, 파싱, I/O 등 recoverable runtime path → 반드시 `Result` 전파 또는 graceful fallback
  - bare unwrap→expect 기계적 치환 금지
- **Files**: src-tauri/src/ 전체 production 코드
- **Gate**: cargo test green + 앱 기동 확인
- **Acceptance**: production 코드에서 bare unwrap() 0건

#### A-2. tracing 도입 — subscriber 초기화 + eprintln 교체 {parallel: false}
- **Actions**:

  - 전체 eprintln! → tracing::warn!/error!/info! 전환
- **Files**: Cargo.toml, src-tauri/src/lib.rs, 전체 eprintln 사용 파일
- **Dependencies**: A-1 이후

## Acceptance Criteria

- [ ] production Rust 코드에서 bare unwrap() 0건 (expect with context 허용, 테스트 제외)
- [ ] tracing subscriber 동작 + 구조화 로그 출력
- [ ] 모든 상태값 enum 사용 (TeamRunStatus, TaskStatus — DB TEXT 호환 유지)
- [ ] production 코드에서 1,000줄 초과 파일 0개 (Rust + TS/TSX)
- [ ] production TS/TSX에서 any 타입 0건 (테스트 제외)
- [ ] 기존 테스트 전체 통과
- [ ] 빌드 에러 0건
- [ ] 매 슬라이스 후 cargo test + npm test green

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| 스토어 분리 시 상태 동기화 버그 | 매 슬라이스 후 수동 smoke test |
| enum 도입 시 wire format 호환 | DB TEXT 유지 + Tauri 직렬화 보존 |
| Rust 파일 분할 시 pub 가시성 | 컴파일 에러로 즉시 감지 |
| CSS 분리 시 스타일 누락 | 빌드 후 전체 화면 육안 검증 |
| clone 최적화 과잉 적용 | 증거 기반으로만 적용 |
