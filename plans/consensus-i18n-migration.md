## Agreed Plan

### Architecture
- **Locale**: i18next language (`ko` | `en`)
- **Namespace**: feature-based (`common`, `glossary`, `settings`, `onboarding`, `agent`, `chat`, `network`, `vault`, `prompts`)
- **Theme**: i18next `context` (`_classic` / `_org`) for glossary terms
- **Translation loading**: static bundled imports (no HTTP fetch ‚ÄĒ Tauri CSP)
- **Non-React layers**: `i18n.t()` singleton (stores/services)
- **i18next config**: `fallbackLng: "ko"`, `returnNull: false`, custom `missingKeyHandler` for dev console warnings
- **Rust resources**: locale-specific directories (`resources/ko/`, `resources/en/`), `TOOL_CONFIG.json` shared (language-neutral)
- **Rust resource loading**: `include_str!` with `match locale` pattern (compile-time, no runtime file reads)
- **Default manager identity**: resolved via `is_default` flag, NOT hardcoded folder name
- **`name` field**: real localized text seeded per locale, editable user data afterward
- **Existing installs**: `Žß§Žčąž†Ä` folder kept as-is, no rename. Resolved by `is_default`
- **New installs**: `manager` folder as stable internal ID

## Implementation Steps

### Step 1: i18n Infrastructure Setup `{parallel: false}`
- Actions:
  - `npm install i18next react-i18next`
  - Create `src/i18n/` directory
  - `src/i18n/config.ts`: i18next init with:
    - `initImmediate: false` (sync)
    - `fallbackLng: "ko"`
    - `returnNull: false`
    - Custom `missingKeyHandler`: `console.warn` in dev (`import.meta.env.DEV`), no-op in prod
    - All namespaces eager-loaded at startup
    - `lng` = localStorage `locale` || `"ko"`
  - `src/i18n/index.ts`: singleton export
  - `main.tsx`: `import './i18n'` before render
  - `settingsStore.ts`: add `locale: "ko" | "en"`, `setLocale()` action, localStorage sync, `i18n.changeLanguage()` call
- Files: `package.json`, `src/i18n/config.ts`, `src/i18n/index.ts`, `src/main.tsx`, `src/stores/settingsStore.ts`

### Step 2: Translation JSON Files `{parallel: true, after Step 1}`
- Actions:
  - `src/i18n/locales/ko/` ‚ÄĒ 9 namespace JSON files:
    - `common.json`: buttons, generic actions, loading/error states, date labels
    - `glossary.json`: theme-sensitive nouns with context keys (`agent_classic`, `agent_org`, `defaultTitle_classic`, `defaultTitle_org`, `hire_classic`, `hire_org` etc.)
    - `settings.json`: settings modal tabs, form labels, hints
    - `onboarding.json`: welcome screen, language/theme selection
    - `agent.json`: agent editor, persona tabs, export/import
    - `chat.json`: input placeholders, message actions, conversation switcher
    - `network.json`: P2P network settings
    - `vault.json`: vault/memory UI
    - `prompts.json`: system prompt scaffolding strings (summary prompt, workspace section, conversation summary header, tool descriptions)
  - `src/i18n/locales/en/` ‚ÄĒ same structure, English translations
  - Key format: shallow nested within namespace (e.g. `tabs.general` in `settings` ns, `input.placeholder.idle` in `chat` ns)
  - Long-form prompt templates: locale-specific `.md` resources in Rust bundle, NOT in JSON
- Files: `src/i18n/locales/ko/*.json`, `src/i18n/locales/en/*.json`

### Step 6b: Existing Install Locale-Aware Refresh `{parallel: false, after Step 6a}`
- Actions:
  - Policy: refresh happens at BOTH startup and locale change
  - `refresh_default_manager_persona` becomes locale-aware:
    - Accept locale parameter
    - For each persona file (`IDENTITY.md`, `SOUL.md`, `USER.md`, `AGENTS.md`):
      - Compare current file content against bundled-default comparison set:
        - **(a)** Legacy pre-i18n default persona contents (current hardcoded Korean defaults before this migration)
        - **(b)** New ko bundled defaults
        - **(c)** New en bundled defaults
      - If file matches any of (a)/(b)/(c) ‚Üí overwrite with current locale's bundled version
      - If file has been customized ‚Üí preserve as-is
  - Default manager resolved by `is_default` flag, tolerating both `Žß§Žčąž†Ä` and `manager` folder names
  - On locale change from Settings: trigger `refresh_default_manager_persona` with new locale
- Files: `agent_commands.rs`, `settingsStore.ts` (setLocale triggers refresh)

## Risks

- Missing i18n keys ‚Üí mitigated by `fallbackLng: "ko"` + dev `missingKeyHandler` + test key coverage
- English system prompts affect LLM behavior ‚Üí separate prompt quality verification
- Startup order change ‚Üí thorough testing of fresh install and existing install paths
- Persona refresh on locale change ‚Üí compare-against-bundled-defaults strategy ensures no custom content loss
