# Vault UI - Consensus Plan

> Generated by /cplan (Claude + Codex consensus)
> Consensus reached: Round 2
> Date: 2026-03-18

## Requirement

Obsidian 호환 볼트 시스템의 전체 프론트엔드 UI 구현. 백엔드는 100% 완성 (CRUD, 위키링크, 그래프, 검색, 파일 감시, 마이그레이션). UI 컴포넌트 16개 신규 생성.

## Agreed Plan

### Architecture

- **Navigation**: 전용 `navigationStore.ts`에 `mainView: "chat" | "network" | "vault"` 열거형, localStorage 영속화
- **Layout**: VaultPanel → 메인 영역 교체 (NetworkPanel 패턴), 3존 적응형 (NoteListPane 300px + DetailPane/GraphPane flex:1)
- **Graph**: D3-force (d3-force, d3-selection, d3-zoom, d3-drag) SVG 렌더링
- **Markdown**: 기존 react-markdown 활용 + 위키링크 커스텀 플러그인
- **Agent Context**: agentStore.selectedAgentId 기본값, vault 내 에이전트 드롭다운으로 변경 가능. 앱 다른 곳에서 에이전트 변경 시 vault는 수동 드롭다운 선택 유지 (auto-follow 하지 않음)
- **Search**: 검색 결과가 노트 목록을 대체 (별도 모드, 검색 해제 시 원래 목록 복귀)

### Key Decisions

1. `navigationStore.ts` 전용 스토어로 mainView 관리 (settingsStore와 분리)
2. `networkStore.networkViewActive` 제거 → mainView 사용
3. vault agent context = 수동 드롭다운 (앱 에이전트 변경에 auto-follow 안 함)
4. 검색 결과 = 노트 목록 대체 모드

## Implementation Steps

### Step 1. Navigation + State Contract `{parallel: false}`
- Actions:
  - `navigationStore.ts` 생성: `mainView` 열거형 + localStorage 영속화
  - MainLayout.tsx 리팩터: boolean 체크 → mainView switch
  - Sidebar.tsx 리팩터: 네트워크/볼트/채팅 전환을 mainView로 통합
  - `networkStore.networkViewActive` 제거, mainView로 대체
  - d3-force 패키지 설치
  - App.css에 볼트 CSS 변수 추가
- Files: `src/stores/navigationStore.ts` (new), `src/stores/networkStore.ts`, `src/components/layout/MainLayout.tsx`, `src/components/layout/Sidebar.tsx`, `src/App.css`, `package.json`
- Dependencies: 없음

### Step 2. VaultPanel + VaultHeader + VaultEmptyState `{parallel: false}`
- Actions:
  - VaultPanel: 루트 컨테이너, local state (viewMode, isEditing, showCreateDialog)
  - VaultPanel mounts → loadNotes(selectedAgentId)
  - VaultHeader: agent dropdown (에이전트 선택 기본값 → 수동 변경 가능), 목록/그래프 토글, 새 노트/Obsidian 버튼
  - VaultEmptyState: 중앙 안내 텍스트
- Files: `src/components/vault/VaultPanel.tsx`, `VaultHeader.tsx`, `VaultEmptyState.tsx`
- Dependencies: Step 1

### Step 3. NoteListPane + NoteSearchBar + NoteFilterBar + NoteListItem `{parallel: true}`
- Actions:
  - NoteListPane: 좌측 300px, 검색 + 필터 + 노트 목록, 로딩 스켈레톤
  - NoteSearchBar: 입력 + 스코프 (전체/내 노트/공유), 300ms 디바운스, Escape 해제
  - NoteFilterBar: 카테고리 칩 (지식=#6366f1, 결정=#f59e0b, 대화=#10b981, 회고=#8b5cf6), 태그 드롭다운
  - NoteListItem: 제목, 2줄 미리보기, 카테고리 배지, 신뢰도 바, 날짜, shared 배지
  - 검색 모드: 결과가 노트 목록 대체, 클리어 시 원래 목록 복귀
- Files: `src/components/vault/NoteListPane.tsx`, `NoteSearchBar.tsx`, `NoteFilterBar.tsx`, `NoteListItem.tsx`
- Dependencies: Step 2

### Step 4. NoteDetailPane + NoteMetadataBar + NoteContent + BacklinksSection `{parallel: true}`
- Actions:
  - NoteDetailPane: 우측 flex:1, 노트 읽기 뷰, 편집/삭제/Obsidian 열기 버튼
  - NoteMetadataBar: 에이전트 배지, 카테고리 배지, 신뢰도 바, 태그 칩 (클릭→필터), 타임스탬프
  - NoteContent: react-markdown + 위키링크 커스텀 렌더러 (resolved=primary색, broken=빨간 점선)
  - BacklinksSection: 접이식 백링크 목록, 클릭→노트 이동
  - 로딩/에러 상태 표시
- Files: `src/components/vault/NoteDetailPane.tsx`, `NoteMetadataBar.tsx`, `NoteContent.tsx`, `BacklinksSection.tsx`
- Dependencies: Step 2

### Step 5. NoteEditor + NoteEditorToolbar + CreateNoteDialog `{parallel: true}`
- Actions:
  - NoteEditor: 마크다운 textarea + 50/50 split 미리보기, Ctrl+S 저장, dirty tracking
  - NoteEditorToolbar: Bold/Italic/H1/H2/Wikilink 아이콘 버튼
  - CreateNoteDialog: 모달 (제목, 분류 radio, 범위 radio, 태그, 내용), 에이전트 자동 설정
  - 미저장 변경 경고 (취소 시)
- Files: `src/components/vault/NoteEditor.tsx`, `NoteEditorToolbar.tsx`, `CreateNoteDialog.tsx`
- Dependencies: Step 2

### Step 6. GraphPane + GraphCanvas `{parallel: true}`
- Actions:
  - GraphPane: 그래프 뷰 컨테이너, 범례 오버레이, 노드/엣지 카운트
  - GraphCanvas: D3-force SVG (ref + useEffect 패턴)
  - 노드: noteType색상, confidence크기(8-20px), shared=점선
  - 엣지: 화살표, var(--border)색
  - Hover: 확대+하이라이트, Click: 목록+상세로 전환, Drag/Zoom/Pan
- Files: `src/components/vault/GraphPane.tsx`, `GraphCanvas.tsx`
- Dependencies: Step 1

### Step 7. Integration + CSS `{parallel: false}`
- Actions:
  - 전체 컴포넌트 연결 검증
  - App.css 볼트 섹션 완성 (~400줄)
  - 위키링크 스타일 (resolved/broken)
  - 반응형: >=1200px 3패널, 900-1200px 2패널
  - tsc --noEmit 통과 확인
- Files: `src/App.css`, 기존 컴포넌트 미세 조정
- Dependencies: Steps 3-6

## Acceptance Criteria

- [ ] mainView 열거형 기반 네비게이션 (chat/network/vault 전환)
- [ ] 사이드바 볼트 아이콘 → VaultPanel 표시
- [ ] 에이전트 드롭다운으로 vault 스코프 변경
- [ ] 노트 목록 필터링 (에이전트, 카테고리, 태그)
- [ ] 전문 검색 (디바운스, 스코프, 결과 목록 대체)
- [ ] 노트 선택 → 상세 뷰 (메타데이터 + 마크다운)
- [ ] 위키링크 클릭 → 노트 이동 (resolved/broken 스타일 구분)
- [ ] 백링크 표시 및 네비게이션
- [ ] 노트 생성 다이얼로그
- [ ] 노트 편집 (에디터 + 실시간 미리보기 + 저장/취소)
- [ ] 그래프 뷰 (D3-force, 클릭/드래그/줌)
- [ ] Obsidian에서 열기
- [ ] 로딩/에러/더티 상태 UX

## Codex Final Assessment

**Strengths:**
- mainView 모델이 모드 충돌 방지
- 에이전트 컨텍스트가 명시적이고 주요 워크플로우에 적용
- 상태 계약이 명확하여 병렬 작업 현실적
- 로딩/에러/더티 상태 UX 포함
- 컴포넌트 분해 깔끔, 시퀀싱이 실제 프론트엔드 리스크에 맞음

**Overall:** v2가 Round 1의 구조적 문제를 수정하고 병렬 작업에 필요한 결정을 올바르게 추가함. 승인.
